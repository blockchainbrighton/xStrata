The Protocol Logic

XStrata isn't just storage; it's a cryptographic verification engine.

01

Fragmentation

Files are sliced into 64KB buffers. A Merkle Tree is generated client-side to create a unique root-hash for the entire file.

02

Inscription

Chunks are uploaded individually or in batches. The smart contract validates every byte against the proof, ensuring no data corruption occurs.

03

Sealing & Minting

Once all chunks are received, the seal-inscription function locks the data forever and mints the xstrata-inscription NFT to your wallet.

;; Core Inscription Logic
(define-public (seal-inscription (hash (buff 32)))
  (let ((pending-key { hash: hash, owner: tx-sender })
        (meta (unwrap! (map-get? PendingInscriptions pending-key) ERR-NOT-FOUND))
        (id (var-get next-id)))
    
    ;; 1. Verify Completion
    (asserts! (is-eq (get received-count meta) (get chunk-count meta)) ERR-NOT-COMPLETE)
    
    ;; 2. SIP-009: Mint the NFT
    (try! (nft-mint? xstrata-inscription id tx-sender))

    ;; 3. Lock Data Immutably
    (map-insert Inscriptions id {
        owner: tx-sender,
        sealed: true,
        merkle-root: hash,
        data-hash: (get context meta)
    })
    
    (ok id)))
